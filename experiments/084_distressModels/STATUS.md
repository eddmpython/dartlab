# 084_distressModels — 기업 부실징후 포착 모델 검증

## 목적
세계적으로 검증된 부실 예측 모델들을 DART 데이터로 계산하고, 한국 시장에서의 유효성을 검증한다.

## 핵심 발견 종합

### 엔진 흡수 우선순위

| 순위 | 모델/기능 | 위치 | 근거 |
|------|----------|------|------|
| 1 | **Ohlson O-Score** | `ratios.py` | 금융업 포함 100% 커버, 확률 해석 |
| 2 | **Altman Z''-Score** | `ratios.py` | Z-Score 불가 금융업 전수 커버 |
| 3 | **앙상블 부실 점수** | `insight/distress.py` 신규 | 5단계 distressLevel |
| 4 | **시계열 악화 탐지** | `anomaly.py` 확장 | 연속적자, ICR<1, CCC 확대 |
| 5 | **감사 Red Flag** | `grading.py` 확장 | 감사인 교체, 보수 급변, 소송 |
| 6 | **관리종목 자동 점검** | `insight/listing.py` 신규 | 자본잠식/매출/연속적자 |
| 7 | Zmijewski X-Score | `ratios.py` | 간결, 금융업 보정 필요 |
| 8 | Springate S-Score | `ratios.py` | cutoff 하향 필요 |
| - | Taffler Z-Score | 보류 | 한국 변별력 부족 |

### 업종 보정 필수 사항
- **금융업**: Z-Score 불가, X-Score 왜곡, ICR<1 구조적, CCC 불가 → 별도 처리
- **지주회사**: DSO 극단값 (LG 9766일, SK 1174일) → DSO/CCC 비적용
- **바이오**: DIO 극단값 (셀트리온 987일) → DIO 별도 기준
- **반도체**: CCC 구조적 장기 (SK하이닉스 222일) → 업종 벤치마크 대비 판단

### Backtesting 한계
- 실제 관리종목/상폐 기업은 DART 데이터 비공개로 직접 테스트 불가
- "과거 위기 이력" 기업은 대부분 이미 재무 회복 → 현재 시점 모델로는 safe 판정
- 금융업·지주회사 false positive 해결 위해 업종별 cutoff 분리 필수
