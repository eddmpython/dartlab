# 067 — 한국어 임베딩 vs 구조 분해 항목 매칭

**상태**: 완료
**실험일**: 2026-03-18

## 핵심 결론

**임베딩(ko-sroberta)은 테이블 항목 매칭에 부적합하다.** 구조 분해 매칭이 모든 면에서 우월하다.

| 방법 | Precision | Recall | F1 | 속도 (283종목) |
|------|-----------|--------|------|---------------|
| Fuzzy (SequenceMatcher) | 분리 불가 | 분리 불가 | — | < 1초 |
| 임베딩 (ko-sroberta) | 분리 불가 | 분리 불가 | — | ~38분 |
| 정규화 + 포함 매칭 | 100% | 78% | 88% | < 1초 |
| **구조 분해 매칭** | **100%** | **90%** | **95%** | **~4분** |

## 임베딩이 실패하는 이유

1. **의미적 유사 ≠ 매칭 필요**: "현금배당수익률" vs "현금배당성향"은 cosine=0.91인데 다른 항목
2. **단일 threshold 불가**: positive min=0.65 < negative max=0.72 → 겹침 구간 존재
3. **속도 비현실적**: 14개 항목 인코딩에 0.56초, 283종목 × 200항목 = 38분
4. **정규화 후에도 분리 불가**: 정규화 + 임베딩 조합도 gap=-0.06

## 구조 분해 매칭의 핵심 아이디어

항목명을 3개 층으로 분해:
```
원본: (연결)당기순이익(백만원)(주22)
  → core: 당기순이익
  → qualifier: [연결]  (distinct → 구분 필요)
  → annotation: [(백만원), (주22)]  (제거)
```

매칭 규칙 (우선순위):
1. core exact + qualifier 호환 → 매칭
2. core 포함관계 (80% 이상) → 매칭
3. core 소문자+특수문자 제거 후 비교 → 매칭
4. core fuzzy ratio >= 0.90 → 매칭

## 남은 한계 (FN 1개)

- `당기순이익` vs `(연결)당기순이익`: 한쪽에만 distinct qualifier가 있는 경우
- 이것은 실제로 컨텍스트 의존적 (같은 테이블 내 연결/별도 구분 여부에 따라 다름)
- 해결 방안: 같은 테이블에 `(별도)당기순이익`이 없으면 `(연결)당기순이익` = `당기순이익`으로 처리
