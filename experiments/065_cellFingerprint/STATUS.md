# 065 셀 타입 핑거프린트

## 상태: 완료

## 결과 요약

### 핵심 수치
- **8개 종목, 316 topics, 같은 구조 판정 99.1%** (313/316)
- "다른 구조" 3개: mdna(2종목), subsequentEvents(1종목)
- 합성 테스트 4/4 정확, 실제 데이터 검증 완료

### 핑거프린트 방법
각 셀을 4가지 타입으로 분류 → 열별 주요 타입으로 시그니처 생성:
- N: 숫자 (1,234 / △500 / **"-" 포함**)
- T: 텍스트
- D: 날짜
- E: 빈 셀

### 핵심 발견
1. **"-"를 N으로 재분류**하면 sparse 테이블 판별 정확도 급상승 (0.47 → 1.0)
2. **열 프로필 + 위치 매칭** 가중 결합이 최적 (0.4 프로필 + 0.6 위치)
3. 이력형(companyHistory)은 기간별 데이터 차이가 커서 핑거프린트가 자연스럽게 낮음 → 정확한 판별
4. boardOfDirectors/executivePay의 초기 "다른 구조" 판정은 과거 기간(2016~)에서 구조 변경이 있었기 때문 → 최근 기간만 보면 100% 매칭

### 장단점

**장점:**
- 헤더 텍스트에 독립적 ("인원수" vs "인원(명)" 무관)
- 이력형(날짜+텍스트) vs 정형(텍스트+숫자) 자동 구분
- 구현 단순, 외부 의존성 없음

**단점:**
- 같은 시그니처의 다른 의미 테이블 구분 불가 (TNT=보수 vs TNT=사업목적)
- 대체재가 아닌 보완재 — 현행 헤더 정규화와 결합 필요

### 적용 제안
현행 `_groupHeader()` + 핑거프린트 2단계:
1. 헤더 매칭 성공 + 핑거프린트 ≥ 0.5 → 같은 구조
2. 헤더 매칭 실패 + 핑거프린트 ≥ 0.8 → 같은 구조 (보완)
3. 핑거프린트 < 0.5 → 다른 구조 (거부)

## 추가 실험: 교차값 기반 행 매칭 (2026-03-18)

### 질문
multi_year 테이블에서 "2024당기 == 2025전기" 교차값으로 항목명 불일치를 매칭할 수 있는가?

### 001 결과: 가설 기각

10종목 x 4 topic(dividend, salesOrder, rawMaterial, riskDerivative):
- **multi_year에서 항목명 불일치: 0건** — DART는 기간마다 항목명 동일 유지
- salesOrder, rawMaterial, riskDerivative는 multi_year가 아님 (kv/matrix)
- 교차값 체인은 multi_year에만 존재 → 적용 범위 없음

### 002 결과: 37% 실패의 실제 분포

10종목 8,013 table 블록:

| 원인 | 비율 | 성격 |
|------|------|------|
| 성공 | 38.5% | - |
| overlap_low (이력형) | 24.8% | 정상 스킵 |
| list_skip (목록형) | 17.8% | 정상 스킵 |
| sparse | 10.9% | 개선 가능 |
| no_items | 8.0% | 일부 개선 가능 |

37% 실패 중 43%는 수평화 부적합 (정상 스킵). 진짜 실패율은 약 19%.

### 003 결과: 값 fingerprint — false positive 과다

730 sparse 블록에서 656건 매칭 후보 발견 (89.9%) — 하지만 대부분 가짜 매칭:
- 날짜 표기 차이, 기수 포함 항목명, 위원회/회사명 변경 등
- 자동 매칭에 부적합 (false positive 높음)

### 결론

1. 교차값 접근의 전제가 틀림 — multi_year에서 항목명 불일치 자체가 없음
2. 37% 실패 중 43%는 수평화해서는 안 되는 테이블
3. 값 fingerprint는 ambiguity 높아 자동 매칭 부적합
4. 62.9% → 70%대는 sparse/no_items 일부 해결로 가능, 80%+ 는 구조적 불가
