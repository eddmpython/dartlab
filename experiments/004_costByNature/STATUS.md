## 발견 사항

### 데이터 위치
- section_title: 주로 "연결재무제표 주석", "재무제표 주석", "III. 재무에 관한 사항"
- 주석 번호: 종목마다 다름 (19~34번 범위)
- 키워드: "비용의 성격" — 모든 종목에서 동일

### 존재 여부 (267개 종목 전수 조사)
- **171/229 종목에서 발견** (보고서 있는 종목 기준 74.7%)
- 38개: 보고서 자체 없음 (스팩, 신규상장 등)
- 58개: 비용의 성격 미공시 (금융업, 리츠, 지주회사 등)
- K-IFRS 의무공시 항목이지만 **금융업은 예외**

### 테이블 유형 (2가지)
1. **inline (83개, 49%)**: `| 구분 | 당기 | 전기 |` — 한 테이블에 당기/전기 병렬
2. **split (88개, 51%)**: `당기 | (단위:백만원)` → 데이터 → `전기 | (단위:백만원)` → 데이터 — 기간별 별도 블록

### 섹션 번호 형식 (3가지)
1. `N. 비용의 성격별 분류` — 가장 흔함
2. `(N) 비용의 성격별 분류` — 하위 번호
3. 번호 없이 키워드만 — 테이블 내 포함

### 기간 표기 형식 (다양)
- "당기" / "전기"
- "당 기" / "전 기" (공백)
- "제 N(당) 기" / "제 N(전) 기"
- "(당기)" / "(전기)"
- "① 당기" / "② 전기"

### 계정 항목 통계
- 평균: 13.4개
- 최소: 1개, 최대: 38개
- 공통 계정: 재고자산변동, 원재료사용, 종업원급여, 감가상각비, 무형자산상각, 기타비용
- 종목별 추가: 외주가공비, 지급수수료, 운반비, 광고선전비, 동력비, 수선비 등

### 파싱 핵심 로직
1. **섹션 탐색**: 번호 패턴 + 키워드 폴백 (`_isPeriodLabel` 통합 함수)
2. **기간 판별**: `_isDanggi` / `_isJeongi` — "제N(당)기" 등 다양한 표기 통합 처리
3. **파싱 순서**: inline → split → multiCol (가장 매칭 빈도 높은 순)
4. **단위 처리**: `detectUnit`로 백만원/천원/원 자동 감지
