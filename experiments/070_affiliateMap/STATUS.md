## 설계 결정 로그

### Union-Find → seed-based 전환
- Union-Find는 경영참여 체인으로 메가컴포넌트 생성 (삼성→KCC→현대 체인)
- **seed + 방향 기반 확장**이 더 정확: A→B 경영참여 = B는 A의 그룹
- well-known dict 80+개 종목코드 → 그룹 매핑으로 seed

### 5단계 분류 파이프라인 (012/013)
1. **Phase 0**: docs 계열회사 현황 ground truth lock (159)
2. **Phase 1**: well-known 확대판 lock (+46)
3. **Phase 2**: 경영참여 방향 확장 (known 그룹 차단, unknown 자유 확장)
4. **Phase 3**: majorHolder 법인주주 (known 20%+, unknown 무제한)
5. **Phase 4**: 공유 개인주주 (기존 그룹에 2+ anchor 있을 때만)
6. **Phase 5**: 이름 키워드 + 나머지 독립

### well-known 자동화 가능성 (014)
- docs + majorHolder 20%+로 수작업의 79.5% 자동 커버
- 나머지 26개(금융지주, HLB, 셀트리온 등)만 수작업 보충 필요
- 패키지 이관 시: 하드코딩 126개 → 자동 추출 + 보충 26개로 80% 축소 가능

### 독립 53%는 정확한 분류, ego는 100% 보강 (016~018)
- 한국 상장사 1,620개 중 대기업집단 소속은 실제로 절반 미만
- 독립 회사는 출자/지분/인적 관계가 공시에 없는 회사 — 독립이 정답
- 키워드 확대/임계값 완화/단순투자 활용 3가지 시도 → 그룹 변경 효과 미미 (최대 -1%p)
- **그러나 사용자 경험은 100% 보강 가능**:
  - 전략 A (단순투자 엣지): 독립 86%가 투자 관계 보유 → ego 뷰에 이웃 추가
  - 전략 C (업종 클러스터): 100% 업종 메타 보유 → 연결 적은 회사에 같은 업종 이웃 보충
  - 결과: 독립 851개 중 1노드 회사 **0개**, 평균 9.9노드
- 공정위 대기업집단 목록 등 외부 데이터 연동 시 추가 분류 가능 (선택적 확장)
