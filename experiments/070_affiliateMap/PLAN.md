# 070_affiliateMap — 한국 기업 관계 지도 기획

## 비전
> DART 공시 데이터로 **한국 상장사 전체의 출자/지분/인적 관계를 하나의 그래프**로 만든다.
> 사용자가 전체를 조감하고, "이 회사랑 이 회사도 관련 있네?"를 발견하는 경험.

## 데이터 소스 (3-Layer)

| Layer | 소스 | 관계 유형 | 노드 | 핵심 가치 |
|-------|------|----------|------|----------|
| **L1 출자** | `investedCompany` | 회사→회사 (출자) | 회사 | 재벌 클러스터, 경영참여/단순투자 |
| **L2 지분** | `majorHolder` | 사람/회사→회사 (지분) | 회사+사람 | 최대주주, 특수관계인, 순환출자 |
| **L3 그룹** | Union-Find + 공유주주 | 자동 탐지 | - | 100% 그룹 분류 |

## 사용자 시나리오

### S1. 전체 조감도
- 전체 상장사 클러스터 맵, 재벌 그룹별 색상
- "삼성이 이렇게 크구나", "이 회사가 여기 그룹이었어?"

### S2. 종목 검색 → 관계 지도
- "삼성전자" 검색 → ego 네트워크 (출자 + 주주 + 순환출자)
- "삼성생명이 최대주주고, 삼성전자가 삼성SDI에 투자하는구나"

### S3. 인물 네트워크
- "이재용" → 연결된 모든 회사, 특수관계인 관계
- "이재용이 여기도 주주였어?"

### S4. 그룹 비교
- 삼성 vs 현대 서브그래프 나란히

## 실험 로드맵

| 실험 | 목표 | 입력 | 출력 |
|------|------|------|------|
| 006 | majorHolder 스캔+엣지 | 978 parquet | 주주→회사 엣지 |
| 007 | 3-Layer 통합+그룹 100% | L1+L2 | 완전 그래프 |
| 008 | UI-ready JSON | 완전 그래프 | overview/full/ego JSON |
| 009 | 시나리오 검증 | JSON | S1~S4 테스트 통과 |

## 최종 산출물

```
output/
├── overview.json     # 전체 조감도 (그룹=슈퍼노드)
├── full.json         # 전체 노드+엣지 (회사+사람, 그룹 100%)
├── ego_{code}.json   # 종목별 ego 서브그래프
└── cycles.json       # 순환출자 경로
```
