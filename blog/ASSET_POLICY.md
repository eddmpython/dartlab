# Asset Policy

블로그 자산은 각 포스트 폴더의 `assets/`에서 관리하고, 빌드 시 최종 `/blog/assets/*` URL로 평탄 복사한다.

## 위치 규칙

- 자산은 반드시 해당 포스트 폴더 아래 `assets/`에 둔다.
- 루트 `blog/assets/`는 사용하지 않는다.
- 본문 참조는 항상 `./assets/...` 상대경로를 쓴다.

## 파일명 규칙

- 모든 자산 파일명은 포스트 번호 접두사를 붙인다.
  - 예: `012-depreciation-flow-map.svg`
- 다른 포스트와 같은 파일명을 쓰면 빌드가 실패한다.

## SVG 기준

- 심화 글 기본값: `5개`
- 큰 시리즈 글: `6개`까지 허용
- 최소선: `4개`

권장 구성:

- 히어로 구조도 1개
- 개념 연결도 1개
- 비교/리스크 도표 1~2개
- 체크리스트 또는 타임라인 1개

## 품질 규칙

- SVG 텍스트 안의 `&`, `<`, `>`는 XML escape를 지킨다.
- alt 문구는 설명형으로 쓴다.
- `image`, `diagram`, `graphic` 같은 일반어 반복은 피한다.
- 모바일에서도 읽히는 글자 크기와 여백을 유지한다.

## 검증 체크리스트

1. 본문 자산 경로가 `./assets/...`인지 확인한다.
2. 파일명 접두사가 포스트 번호와 일치하는지 확인한다.
3. `npm run build` 후 `landing/build/blog/assets`에 복사됐는지 확인한다.
4. 자산 파일명 충돌이 없는지 확인한다.
5. 접근성 경고와 SVG XML 문법 오류가 없는지 확인한다.
