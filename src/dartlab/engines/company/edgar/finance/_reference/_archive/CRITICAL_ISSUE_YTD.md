## 문제 상세

### GOOG 2024 Revenue (frame=null)

**원본 데이터**:
```
Q1: $80.54B (1개 값만 존재)
Q2: $144.39B, $165.28B (2개 YTD 값)
Q3: $221.08B, $253.55B (2개 YTD 값)
```

**현재 로직 (min 선택)**:
```
Q1: $80.54B ✅
Q2: $144.39B ❌ (실제는 $84.74B여야 함)
Q3: $221.08B ❌ (실제는 $88.27B여야 함)
```

**정답 (YTD 역산)**:
```
Q1: $80.54B
Q2: $165.28B - $80.54B = $84.74B
Q3: $253.55B - $165.28B = $88.27B
```

## 근본 원인

### 현재 가정 (잘못됨)
> "EDGAR는 각 분기마다 개별 값과 YTD 값을 함께 제공한다"
> → `min()` 선택으로 개별 값 추출 가능

### 실제 상황
> "일부 기업은 **개별 분기 값을 전혀 제공하지 않고** YTD만 제공한다"
> → `min()` 선택해도 여전히 YTD

### 타입 B: YTD만 제공 (예: GOOG)

```
Q3 값들: $221.08B (YTD), $253.55B (YTD)

현재: min() 선택 → $144.39B, $221.08B ❌
정답: YTD 역산 → $84.74B, $88.27B ✅
```

## 해결 방안

### 방법 1: YTD 자동 감지 및 역산 (권장)

```python
# Q2 처리
q2_val = min(q2_values)
q1_val = min(q1_values)

# Q2가 Q1의 1.5배 이상이면 YTD로 간주
if q2_val > q1_val * 1.5:
    # YTD 역산
    q2_individual = max(q2_values) - q1_val
    final_q2 = q2_individual
else:
    # 개별 값
    final_q2 = q2_val

# Q3도 동일 로직
```

**장점**:
- 타입 A, B 모두 자동 처리
- 휴리스틱 기반이지만 높은 정확도

**단점**:
- 1.5배 임계값이 자의적 (실제로 성장률 150%일 수도 있음)
- NVDA처럼 급성장 기업은 오판 가능

### 방법 2: 모든 Q2/Q3를 YTD로 간주하고 역산

```python
# Q2/Q3는 무조건 max 선택 후 YTD 역산
q2_ytd = max(q2_values)
q2_individual = q2_ytd - q1_val

q3_ytd = max(q3_values)
q3_individual = q3_ytd - q2_ytd
```

**장점**:
- 타입 B에서 100% 정확
- 로직 단순

**단점**:
- 타입 A에서 YTD와 개별이 섞여 있을 때 부정확
- max 선택이 항상 YTD라는 보장 없음

### 방법 3: 검증 로직 추가

```python
# 1. min 선택
q2_min = min(q2_values)

# 2. 검증: Q2 < Q1 * 2 (정상 범위)
if q2_min > q1 * 1.8:
    # YTD로 의심 → max 선택 후 역산
    q2_ytd = max(q2_values)
    q2_final = q2_ytd - q1
else:
    q2_final = q2_min
```
