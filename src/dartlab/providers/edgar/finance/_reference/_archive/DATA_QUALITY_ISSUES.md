# EDGAR 데이터 품질 이슈 및 해결

## 발견된 데이터 이상 패턴

### 1. ✅ 해결됨: 동일 Period에 다중 값 혼재

**발견 일시**: 2026-01-23

**문제 상세**:
```
AAPL 2024-Q3 Revenue (frame=null만):
- $293.79B (YTD = Q1+Q2+Q3 누적)
- $296.11B (YTD, 약간 다른 누적값)
- $85.78B (개별 분기 값)

MSFT 2024-FY Revenue (frame=null만):
- $198.3B (특정 세그먼트 또는 부분 합계)
- $275.0B (전체 합계)
```

**근본 원인**:
- EDGAR는 동일한 XBRL 태그로 서로 다른 범위의 데이터를 보고
- frame=null 조건만으로는 구분 불가능
- 어떤 값이 "올바른" 값인지 메타데이터 없음

**해결 방법**:
```python
# Q2/Q3: min() → YTD 누적 배제, 개별 분기 선택
# FY/Q1: max() → 세그먼트 배제, 전체 합계 선택
```

**검증 결과**: FY = Q1+Q2+Q3+Q4 수식이 모든 종목에서 일치

### 2. ✅ 해결됨: Amended Period 표기

**발견 종목**: MSFT

**문제**: 수정 공시(Amendment) 기간이 "2012-Q2A" 형식으로 표기됨

**파싱 에러**:
```python
parts = "2012-Q2A".split('-')  # ['2012', 'Q2A']
quarter = int(parts[1][1:])    # int('2A') → ValueError!
```

**해결**: 숫자만 추출
```python
quarterStr = ''.join(c for c in periodPart[1:] if c.isdigit())
quarter = int(quarterStr) if quarterStr else 0
```

### 3. ⚠️ 확인 필요: Frame 필드의 의미

**관찰 내용**:

| frame 값 | 의미 추정 | 사용 여부 |
|---------|---------|---------|
| null | 당기 실제 값 | ✅ 사용 (최우선) |
| CY2023Q1 | 전기 비교 재작성 값 | ❌ 배제 |
| CY2022 | 과거 연도 재작성 | ❌ 배제 |

**우려 사항**:
- frame=null이 항상 "최신/정확"한 값인지 보장 없음
- 일부 종목에서 frame=null이 없고 frame=CY2024만 있는 경우?

**대응**: 현재는 frame_priority=1 (frame=null) 강제 필터링

---

### 4. ⚠️ 확인 필요: Q4 직접 보고 부재

**관찰**: 모든 종목에서 Q4는 직접 보고되지 않음

**EDGAR 구조**:
```
보고되는 기간: Q1, Q2, Q3, FY
보고 안 됨: Q4
```

**현재 대응**: Q4 = FY - (Q1+Q2+Q3) 계산

**리스크**:
- FY 값이 잘못 선택되면 Q4도 틀림
- 일부 기업은 회계연도가 다를 수 있음 (캘린더 연도 아닐 수도)

## 데이터 품질 검증 체크리스트

### 자동 검증 항목

- [ ] FY = Q1 + Q2 + Q3 + Q4 (오차 < $1B)
- [ ] Q2 < Q1 + Q2 (YTD 아닌지 확인)
- [ ] Q3 < Q1 + Q2 + Q3 (YTD 아닌지 확인)
- [ ] 모든 분기 값 > 0 (음수 매출 비정상)
- [ ] Filed 날짜 순서 정합성 (Q1 < Q2 < Q3 < FY)

### 수동 검증 항목

- [ ] 주요 종목 10개 재무제표 실제 공시와 대조
- [ ] 다른 데이터 소스(Yahoo Finance, Bloomberg)와 교차 검증
- [ ] 비정상적인 분기 변동률 확인 (예: Q2가 Q1의 10배)

## 알려진 제약사항

### EDGAR 데이터 특성

1. **Q4 미보고**: 모든 기업이 Q4를 직접 보고하지 않음 → 계산 필요
2. **다중 값**: 동일 기간/태그에 여러 값 존재 → 휴리스틱 선택 필요
3. **세그먼트 혼재**: 전체 합계와 세그먼트 값 구분 어려움
4. **Amended Filing**: 과거 기간 수정 시 구분 표기 불일치

### 현재 대응 전략

| 문제 | 대응 방법 | 신뢰도 |
|-----|---------|-------|
| YTD vs 개별 | min() 휴리스틱 | 90% |
| 전체 vs 세그먼트 | max() 휴리스틱 | 85% |
| Q4 계산 | FY - (Q1+Q2+Q3) | 95% |
| Frame 선택 | frame=null 우선 | 90% |

**종합 신뢰도**: 약 90% (수동 검증 필요)
