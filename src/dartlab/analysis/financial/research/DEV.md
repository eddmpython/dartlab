# research 엔진 — DEV

## 기업분석이란 무엇인가

### 질문

코드를 쓰기 전에, "무엇을 분석하는 것인지" 자체를 정의해야 한다.
dartlab의 기능이 먼저가 아니라 방법론이 먼저 선다.

### 기업분석의 8대 영역

교과서 6권(Palepu, Penman, CFA, Fridson, White, Subramanyam) + 실무 프레임워크(McKinsey, S&P/Moody's, PE 실사)를 교차하면, 기업분석은 8개 영역으로 나뉜다.

```
기업분석(Corporate Analysis)
│
├─ [1] 전략/사업 분석 (Strategy & Business Analysis)
├─ [2] 회계/공시 분석 (Accounting & Disclosure Analysis)
├─ [3] 재무분석 (Financial Analysis)
├─ [4] 전망분석 (Prospective Analysis)
├─ [5] 가치평가 (Valuation)
├─ [6] 리스크 분석 (Risk Analysis)
├─ [7] 비교분석 (Comparative Analysis) ← 모든 단계에 횡단 적용
└─ [8] 경제/산업 분석 (Macro & Industry Analysis) ← 외부 환경
```

### 흐름

```
[8] 경제/산업 분석 (외부 환경, Top-down 출발점)
    ↓
[1] 전략/사업 분석 (이 기업은 무엇이고, 왜 돈을 버는가)
    ↓
[2] 회계/공시 분석 (숫자를 믿을 수 있는가)
    ↓
[3] 재무분석 (과거에 무엇이 일어났는가)
    ↓
[4] 전망분석 (앞으로 무엇이 일어날 것인가)
    ↓
[5] 가치평가 (그래서 이 기업의 가치는 얼마인가)
    ↓
[6] 리스크 분석 (무엇이 잘못될 수 있는가)

    ↔ [7] 비교분석 (모든 단계에 횡단 적용)
```

### 관통 축 (렌즈)

| 축 | 설명 |
|---|---|
| 정량 vs 정성 | 숫자 기반 vs 판단/서술 기반 |
| 내부 vs 외부 | 기업 내부 데이터 vs 외부 환경 |
| 과거 vs 미래 | 실적 기반 vs 추정 기반 |
| 절대 vs 상대 | 기업 단독 vs 비교 |
| 재무 vs 비재무 | 재무제표 기반 vs 비재무 정보 |

## 8대 영역 상세

### [1] 전략/사업 분석

정성분석이 주축. 숫자 이전에 "이 회사가 무엇을 하고, 왜 돈을 버는가"를 이해하는 단계.

```
[1] 전략/사업 분석
├─ 1.1 사업모델 (수익구조, 비용구조, 가치사슬 위치, 고객 세그먼트)
├─ 1.2 경쟁우위/해자 (Porter 5 Forces, Morningstar 해자, SWOT, VRIO)
├─ 1.3 경영진/리더십 (이력, 보상구조, 자본배분 트랙레코드, 내부자 거래)
├─ 1.4 지배구조 (이사회, 지분구조, 관계사 거래, 감사 품질, 주주환원)
├─ 1.5 ESG (환경, 사회, 지배구조 — 1.4와 일부 중첩)
└─ 1.6 공시 텍스트/서술형 분석 (MD&A, 리스크 팩터, 톤 분석, 기간간 변화)
```

### [2] 회계/공시 분석

숫자를 믿을 수 있는가? 재무분석 이전에 회계 품질을 검증하는 단계.

```
[2] 회계/공시 분석
├─ 2.1 회계정책 (수익인식, 감가상각, 재고평가, 리스, 충당부채, 정책 변경)
├─ 2.2 이익의 질 (발생액, 지속성, 예측력, 현금이익 괴리, 평활화, Beneish)
├─ 2.3 공시 품질 (적시성, 완전성, 감사의견, 내부회계관리제도)
└─ 2.4 적기경보 (매출-채권 괴리, OCF 괴리, 재고 비정상, 감사인 교체)
```

### [3] 재무분석

과거 숫자를 체계적으로 분해하여 실적의 구조와 추세를 파악하는 단계. 정량분석의 핵심.

```
[3] 재무분석
├─ 3.1 비율분석
│   ├─ 수익성 (매출총이익률, 영업이익률, 순이익률, ROE, ROA, ROIC, EBITDA마진)
│   ├─ 안정성/레버리지 (부채비율, 이자보상배율, Net Debt/EBITDA, 유동비율, 당좌비율)
│   ├─ 효율성 (총자산회전율, DIO, DSO, DPO, CCC)
│   ├─ 성장성 (매출/영업이익/EPS/자산 성장률)
│   └─ 주주가치 (EPS, BPS, DPS, 배당수익률, 배당성향, FCF per share)
├─ 3.2 공통형분석 (수직: 매출=100% IS, 자산=100% BS / 수평: 기준연도 대비)
├─ 3.3 추세분석 (YoY, QoQ, CAGR, 장기 추세선)
├─ 3.4 지수분석 (기준연도=100 정규화)
├─ 3.5 현금흐름분석 (OCF/ICF/FCF 패턴, FCFF/FCFE, 현금전환율, CapEx 강도, 운전자본)
├─ 3.6 분해분석
│   ├─ DuPont 분해 (3-Factor, 5-Factor)
│   ├─ 세그먼트 분해 (사업부문/지역/제품별)
│   ├─ 매출 분해 (물량 × 가격 × Mix)
│   ├─ 비용 분해 (고정비/변동비, 원가 항목별)
│   └─ Bridge 분석 (전기→당기 변동 요인 분해)
├─ 3.7 부실예측 (Altman Z, Ohlson O, Zmijewski, Merton, KMV)
├─ 3.8 이익품질 정량측면 (지속적 vs 일시적, Core Earnings, DOL, DFL)
└─ 3.9 자본구조/자본비용 (WACC, CAPM, EVA)
```

### [4] 전망분석

과거 분석을 기반으로 미래를 예측하는 단계.

```
[4] 전망분석
├─ 4.1 재무제표 추정 (Pro Forma IS/BS/CF, 매출/비용/운전자본/CapEx 예측)
├─ 4.2 성장 모델링 (SGR, IGR, 유기적 vs M&A, S-Curve)
├─ 4.3 민감도 분석 (단일변수, 이변수, 몬테카를로)
└─ 4.4 시나리오 분석 (Base/Bull/Bear, 스트레스 테스트)
```

### [5] 가치평가

1~4단계 분석의 종착점.

```
[5] 가치평가
├─ 5.1 절대 가치평가 (DCF FCFF/FCFE/APV, DDM, Residual Income, 자산기반, 실물옵션)
├─ 5.2 상대 가치평가 (PER/PBR/PSR/EV-EBITDA, 산업특화 멀티플, 거래배수)
└─ 5.3 하이브리드 (SOTP, LBO, Football Field, Reverse DCF)
```

### [6] 리스크 분석

```
[6] 리스크 분석
├─ 6.1 재무 리스크 (유동성, 레버리지, 금리, 환율, 신용)
├─ 6.2 사업 리스크 (매출 집중도, 공급망, 기술 진부화, 규제, 소송/우발부채)
├─ 6.3 시장 리스크 (체계적/비체계적, 변동성, VaR)
├─ 6.4 신용평가 프레임워크 (S&P/Moody's: 사업위험 + 재무위험 + 수정요소)
└─ 6.5 PE/VC 실사 (재무/상업/법률/세무/운영/기술/인력/ESG 실사)
```

### [7] 비교분석

모든 단계에 횡단적으로 적용되는 분석 차원.

```
[7] 비교분석
├─ 7.1 피어 비교 (피어그룹 선정, 재무비율, 멀티플, 운영 KPI)
├─ 7.2 섹터/산업 벤치마크 (평균 대비, 분위수, 리더 vs 래거드)
├─ 7.3 글로벌 비교 (글로벌 피어, 국가리스크 보정, 회계기준 차이)
├─ 7.4 시계열 자기비교 (과거 추세, 과거 밸류에이션 밴드, 가이던스/컨센서스 대비)
└─ 7.5 이벤트 기반 비교 (Pre/Post 이벤트, 동종업체 반응, CAR)
```

### [8] 경제/산업 분석

```
[8] 경제/산업 분석
├─ 8.1 거시경제 (GDP, 금리, 환율, 원자재, 인플레이션, 통화/재정 정책)
├─ 8.2 산업 분석 (수명주기, 산업구조, 집중도, 성장동인, 진입/퇴출 장벽)
├─ 8.3 규제 환경 (산업별 규제, 정부정책, 무역정책, 환경규제)
└─ 8.4 기술/구조 변화 (패러다임 전환, 디지털 전환, 대체 기술, 채택 곡선)
```

## 토론 기록

### 2026-03-26 — 첫 토론

**문제 인식:**
- narrative.py에 27개 함수가 있지만, 이것이 기업분석의 어디를 커버하는지 판단할 기준이 없었다.
- 함수를 몇 개 더 추가하는 것이 아니라, 흔들리지 않는 축을 먼저 세워야 한다.
- 코드가 아니라 방법론이 먼저다.

**실패한 접근:**
1. 밸류에이션/컨센서스까지 섞은 20장 튜토리얼 → 범위 혼재
2. 교과서 갭 분석 후 함수 4개 추가 → 코드에만 집착, 체계 부재

**결정:**
- 기업분석 8대 영역 체계를 먼저 확립
- 이 축 위에 dartlab 현재 기능을 매핑하고, 빈 곳을 식별
- 엔진 명칭 재검토 필요 (research → ?)

**미결:**
- analysis/ 하위 엔진들을 8대 영역에 어떻게 매핑할 것인가
- 현재 `research` 엔진의 정체성과 명칭
- 어디까지가 dartlab의 범위인가 (8개 영역 전부? 일부?)
